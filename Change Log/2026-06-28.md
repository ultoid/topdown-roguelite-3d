# Change Log - 28 Juni 2026

## Animasi & Root Motion
- Memperbarui sistem *Root Motion Removal* di player.gd agar lebih dinamis. Script sekarang mendukung deteksi nama tulang %GeneralSkeleton:Hips dan Skeleton3D:mixamorig_Hips untuk secara otomatis menghapus pergerakan sumbu X dan Z saat berlari/menyerang, mencegah karakter melenceng dari pusat kamera (in-place animation).
- Menyelesaikan isu Found the key outside the animation range dengan mengubah status animasi menjadi *local/built-in* dan mengkonversinya menjadi .tres untuk menghapus flag imported = true, sehingga animasi sepenuhnya dapat diedit di editor Godot tanpa batas.
- Animasi idle karakter telah sukses dipetakan dari global ke struktur library ase/idle.

## Hitbox & Hurtbox
- Merombak total *Hurtbox* pemain. Hurtbox sekarang menggunakan kapsul yang ditempelkan langsung ke tulang karakter menggunakan BoneAttachment3D (Badan, LenganKanan, KakiKiri, dll) agar area tabrakan merespons animasi tubuh karakter secara akurat secara *real-time*.
- *Hitbox* (Area serangan) dari serangan tangan kosong (base attack) sekarang menempel langsung ke tangan (RightHandHitBox) menggunakan BoneAttachment3D, tidak lagi menggunakan proyektil kotak statis di depan pemain.

## Perbaikan Senjata
- Memperbaiki sistem peletakan senjata dinamis (update_equipped_weapon) agar bisa memunculkan senjata (seperti long_sword.tscn) dari file eksternal dengan aman.
- Menyediakan wadah khusus BoneAttachment3D bernama BoneAttachment3D pada GeneralSkeleton:RightHand sebagai *container* murni untuk instansiasi senjata, sehingga memisahkannya dari node hurtbox dan tidak menghapus node penting saat mengganti senjata.

## Perbaikan Hitbox Pedang & Serangan Kombinasi
- Menyelesaikan isu sinkronisasi *hitbox* untuk senajata (`long_sword`) dengan menggunakan perhitungan jarak matematis dari titik pusat `Player` daripada titik `BoneAttachment3D` yang posisinya tidak menentu.
- Menghapus ketergantungan timer statis pada kontrol hitbox, dan mengimplementasikannya melalui integrasi utuh *Method Track* (`activate_weapon_hitbox()` / `deactivate_weapon_hitbox()`) di dalam `AnimationPlayer`.
- Mengubah logika siklus serangan (*combo attack*). Siklus kini tidak diatur oleh waktu, melainkan berjalan secara kontinyu dan hanya di-reset jika pemain menerima hantaman.

## Fisika Permainan & Peta
- Mengaktifkan `CollisionShape3D` pada entitas `red_player_enemy` dan menetapkan koefisien gravitasi konstan untuk musuh pada `enemy.gd`. Ini menyelesaikan *bug* dimana musuh terjatuh menembus peta saat melakukan *spawning* atau pergerakan di `Terrain3D`.
