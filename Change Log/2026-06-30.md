### Sistem Kustomisasi Karakter (Character Customization)
- **Base State (`Global.gd`)**: Kustomisasi `hair` dan `facialhair` sekarang direkam dan dapat dibaca oleh sistem.
- **Dynamic Bone Merging**: Sistem kini dapat meload model kustomisasi `FBX` yang terpisah (misal rambut dan jenggot yang memiliki tulang physics sendiri seperti `hair_dyn` dan `fchr_dyn`), lalu menggabungkan (*merge*) tulang-tulang dinamis tersebut ke `GeneralSkeleton` utama milik *player*.
- **Mesh Reparenting**: Mesh dari bagian kustomisasi secara otomatis dipindahkan (`reparent`) ke dalam `GeneralSkeleton` pemain. Memperbaiki *offset* model yang melorot dengan menerapkan reset `Transform3D.IDENTITY` tepat setelah dipindahkan, memastikan *mesh* rambut/jenggot sejajar sempurna dengan `Base_Head`.
- **Visibility Toggle**: Mesh default (seperti `Cust_Hair` dan `Cust_Beard` bawaan) akan otomatis disembunyikan jika kustomisasi baru dimuat.
- **Realtime UI Preview (`SubViewport`)**: 
  - UI kustomisasi (menu `P`) telah ditambahkan.
  - Memanfaatkan `SubViewport` (dengan `own_world_3d = false`) yang berbagi dunia utama game agar dapat menampilkan *preview* karakter secara langsung dari memori tanpa proses duplikasi objek.
  - Kamera preview ditempatkan tepat di wajah pemain, menggunakan matriks transformasi *player* (`global_transform.basis.z`) agar kamera selalu mengarah ke wajah walau pemain berubah rotasi, dengan posisi sumbu Y setinggi kepala (`2.5`).
- **Offset System & Part Alignment Tool**:
  - Penambahan parameter `position_offset`, `rotation_offset`, dan `scale_offset` ke `customization_data.gd`.
  - Diaplikasikan saat merakit karakter di `player.gd` (ditumpuk pada _Bind Pose_ mesh rambut/jenggot sebelum skinning).
  - Pembuatan _scene_ khusus untuk developer `PartAlignmentTool.tscn` agar bisa mencocokkan rambut/jenggot secara visual di Editor tanpa harus memperbaikinya via software 3D external.
  - **Bugfix (Skinned Mesh Reparenting)**: Perbaikan krusial pada cara sistem memuat rambut/jenggot FBX. Sub-tree FBX kini dipertahankan dan ditambahkan langsung ke `GeneralSkeleton` dengan transform *Identity*, sementara *offset* diaplikasikan langsung pada root FBX. Ini memperbaiki isu di mana skala FBX (0.01) dari Unity rusak akibat reset `Transform3D` yang memicu rambut tumpang-tindih berukuran raksasa.
  - Penambahan *toggle* `apply_offset` di dalam `CustomizationData` agar sistem tidak memaksakan *offset* default (0,0,0) yang berpotensi merusak orientasi asli dari FBX bawaan Unity.
  - **Bugfix (Material)**: Sistem kini otomatis menyalin material dari `Base_Head` ke bagian rambut/jenggot baru yang tidak memiliki material bawaan, menyelesaikan isu rambut terlihat hitam pekat saat di-_load_.

### Script `PlayerVisual.gd` (Baru)
- Ditambahkan script `@tool` baru yang terpasang langsung pada root node `PlayerVisual`.
- **Auto Bone Merge di Editor**: Script berjalan di editor (`@tool`) maupun saat game berjalan. Secara otomatis mendeteksi semua node rambut/jenggot di bawah `Base_Hair` dan `Cust_Beard`, lalu menggabungkan tulang-tulang yang kurang (termasuk physics bones seperti `hair_dyr_01`) ke `GeneralSkeleton` utama.
- **Traversal Rekursif**: Menggunakan traversal rekursif manual (bukan `find_children`) untuk memastikan kompatibilitas penuh dengan sub-scene FBX yang ter-instance di editor.
- **Fallback via Skin Binds**: Jika tidak ditemukan `Skeleton3D` internal (mesh yang diekstrak tanpa skeleton), script membaca data tulang langsung dari `Skin` resource milik `MeshInstance3D` dan menambahkan tulang yang kurang ke `GeneralSkeleton` menggunakan inverse bind matrix.
- **Tujuan**: Memungkinkan workflow editor di mana developer bisa menambahkan FBX rambut/jenggot langsung ke `Base_Hair` dan melihatnya tampil benar, sebagai langkah persiapan sebelum diekstrak ke `Hair_db.tscn` dan `Beard_db.tscn`.
