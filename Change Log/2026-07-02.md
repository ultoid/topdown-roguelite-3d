# Change Log - 2 Juli 2026

## Investigasi: Skin Bind Tulang Tangan (Finger Bones)

### Masalah yang Ditemukan
- Ditemukan error/notifikasi berulang di Godot Output saat runtime:
  ```
  _notification: Skin bind #33 contains named bind 'index_01_l' but Skeleton3D has no bone by that name.
  _notification: Skin bind #34 contains named bind 'index_01_r' but Skeleton3D has no bone by that name.
  _notification: Skin bind #46 contains named bind 'middle_01_l' but Skeleton3D has no bone by that name.
  ... (dan seterusnya untuk semua finger bones)
  ```
- Penyebab: **Perbedaan konvensi penamaan tulang** antara Skin pada mesh dan `Skeleton3D` yang digunakan.
  - **Skin mencari**: `index_01_l`, `index_02_l`, `index_03_l`, `middle_01_l`, dst. *(snake_case)*
  - **Skeleton3D memiliki**: `RightIndexProximal`, `RightIndexIntermediate`, `RightIndexDistal`, `RightMiddleProximal`, dst. *(PascalCase)*

### Status
- ⚠️ **Belum terselesaikan** — Tulang jari tangan pada mesh tidak mau menempel ke `Skeleton3D` karena perbedaan nama ini.
- Karena ini hanya `_notification` (bukan error fatal), game masih bisa berjalan, namun animasi jari tidak berfungsi.

### Rencana Penyelesaian (Investigasi Lanjutan)
- [ ] Cari sumber skin bind — apakah dari file `.glb`/`.fbx` atau ter-embed di `.tscn`
- [ ] Putuskan apakah rename dilakukan di sisi Skeleton3D atau di file 3D source (Blender)
- [ ] Pastikan rename tidak merusak animasi yang sudah ada
